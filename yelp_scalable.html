Tailored User Recommendations

THE PROBLEM

As users continuously review locations they visit, an image is formed of what a user looks for in shopping stores and restaurants. Despite this source of data, Yelp fails to recommend users a prefiltered list of businesses based on their previous reviews. Our solution is to help the users get better recommendations and enhance their experience. By creating user profiles made of common key words in their reviews, we can tailor recommendations based on their needs and wants. These tailored recommendations would lead to increased engagement in Yelpâ€™s services. .

SOLUTION

We developed an advanced recommendation system that suggests restaurants users are most likely to enjoy. Our system analyzes user reviews to identify individuals with similar tastes and preferences. By considering the reviews from these like-minded users, we can offer highly personalized and relevant restaurant recommendations.

DATA

The dataset utilized for this project consisted of circa 5.5 million user reviews for shops, restaurants, and other places of business.

TECHNOLOGIES USED

- Cloud Computing Resources: Utilized cloud clusters to handle large datasets efficiently.
- Data Preparation: Employed PySpark functions such as MAP and reduce for data processing.
- Recommendation System:Built using PySpark's minLSH to calculate similarities.
- Recommendation Generation:Based on the similarity values obtained from minLSH.

APPROACH

- Data Preprocessing:
- Convert user ratings to float and create a "label" column using PySpark.
- Tokenize, remove stop words, and lemmatize review text using PySpark RDDs.
- Run preprocessing code on a cluster for efficiency.

- Filtering Users and Businesses:
-Focus on restaurant data only.
- Exclude users with fewer than 20 reviews using PySpark DataFrames.
- Rebuild user profiles using only 4-star and 5-star reviews.
- Extract business profiles from filtered user reviews.

- Review Processing and Aggregation:
- Store reviews with corresponding restaurants in PySpark DataFrames.
- Tokenize, lemmatize, and aggregate reviews to create final profiles.

Implementation

- Load data into PySpark DataFrames.
- Aggregate top words for users and businesses using user-defined functions in PySpark.
- Initially implemented locality-sensitive hashing (LSH) for clustering, then considered fuzzy hashing.
- Chose final method for more reliable results.
- Average rating not used; recommendations based on cosine similarity of TF-IDF vectors.
- Transform dataframes with TF-IDF vectorizer in PySpark.
- Calculate cosine similarity between user and restaurant profiles; return top 5 recommendations.

Results & Evaluation

- Developed a Leave One Out (LOO) evaluation system to predict user preferences based on historical reviews.
- Split each user's reviews into training and test sets: training with all but one review, test with the held-out review.
- Evaluated predictive accuracy using metrics: precision, recall, accuracy, mean average precision (MAP), and normalized discounted cumulative gain (NDCG).
- Achieved a MAP score of 0.153836 and an NDCG score of 0.300671.
